## **五、文件状态流转**
### 5.1 共享文档状态机
```plaintext
[首次上传] → 审批中 → 已发布
                    ↘ 已驳回 → 重提 → 审批中
                    
[已发布] → 创建编辑副本 → 编辑中 → 提交审批 → 审批中 → 已发布(新版本)
                                            ↘ 已驳回 → 编辑中
[已发布] → 移除（管理员）→ 退回归属人个人中心
[草稿] → 删除（仅归属人）→ 回收站 → 恢复 / 永久删除
```

### 5.2 个人中心状态机
```plaintext
新建 → 草稿 → 编辑中 → 提交发布 → 审批中 → 已发布
              ↘ 邀请协同编辑（共享房间）
```

### 5.3 状态说明

| **状态** | **语义** | **共享列表可见**  | **出现位置** |
|---|---|---|---|
| 草稿 | 个人中心创建的未发布文档 | 否  | 个人中心 |
| 编辑中 | 正在编辑的文档（草稿或编辑副本） | 否  | 个人中心 / 共享中心 |
| 审批中 | 提交审批等待处理 | 否  | 审批中心 |
| 已发布 | 审批通过的正式版本 | **是**  | 共享文档 / 个人中心 |
| 已驳回 | 审批被驳回 | 否  | 个人中心 |
| 已删除 | 在回收站中 | 否  | 回收站 |

### **6.3 共享文档**
#### **6.3.1功能说明**
**（1）功能概述**
共享文档是系统的核心页面，左侧为文档导航树（公司层 / 按部门 / 按产品线），右侧显示选中组的子组卡片和文件列表。支持组的创建、编辑、删除、成员管理等操作。
**（2）功能路径**
侧边栏 → 共享文档 → 文档导航树 → 选中组节点 → 右侧内容区
**（3）UI/原型示例（图片访问不了，可参考prototype-v21.html）**

#### **6.3.2文档导航**

| **元素** | **逻辑描述** |
|---|---|
| **页面布局** | 左侧文档导航树（370px，可拖拽调整宽度）+ 右侧内容区 |
| **文档导航树** | 三个顶级分类节点：公司层 / 按部门 / 按产品线，各自可展开子节点。所有组支持无限层级嵌套 |
| **树节点交互** | 鼠标悬停显示 +（创建子组）和 ···（重命名/删除/交接等操作菜单），参考飞书知识库交互模式 |
| **右侧内容区** | 选中顶级分类（公司层/按部门/按产品线）时，显示对应的创建按钮（创建组/创建部门/创建产品线）+ 下级卡片列表；选中具体组后直接显示：1) 子组卡片（网格布局）；2) 本组文件列表（仅 已发布状态文件） |
| **全局搜索** | 搜索文档名称和组名称，结果分"组"和"文档"两区展示，仅展示有权限的结果 |
| **功能详述**  |
| **组织架构层级** | **公司层** |
公司层
（1）定义与定位
公司层是最顶级的文档分类维度，用于存放跨部门/跨产品线的公共文档（如公司制度、规章流程、公告等）。公司层直接下辖组，组支持无限层级嵌套。
（2）可见范围
- 左侧导航树中「公司层」节点对所有登录用户可见
- 公司层下的组及组内文档：仅该组的成员（管理员/可编辑）可见
- 非组成员只能看到「公司层」节点本身和组名列表，看不到组内文档

（3）创建与管理权限
- 创建一级组           --     系统管理员、公司层管理员
- 创建子组           --      系统管理员、公司层管理员（自建组下）
- 重命名              --     系统管理员、公司层管理员（自建组）
- 成员管理           --     系统管理员、公司层管理员（自建组）
- 组设置              --     系统管理员、公司层管理员（自建组）
- 交接负责人        --     系统管理员、公司层管理员（自建组）
- 删除组              --     系统管理员、公司层管理员（自建组）

（4）子组负责人确定方式
- 系统管理员和公司层管理员可以创建组
- 创建人自动成为该组的组负责人
- 组负责人在该组内拥有管理员权限
- 如需更换负责人，通过「公司层节点 ··· 菜单 -> 交接负责人」操作

（5）管理员设置
触发方式：公司层节点 ··· 菜单 -> "管理员设置"。
- 管理员可创建公司层下的组
- 添加管理员 ：飞书成员选择器
- 移除管理员：文字按钮，confirm「确定移除 {姓名} 的公司层管理员权限？」
- 权限：仅系统管理员可以设置管理员

（6）组设置
入口： 选中公司层下的某个组 → 右侧页面标题后「组设置」按钮（仅组管理员可见）
- 审批流配置
  - 默认开启审批，审批模式为依次审批，审批人默认为组负责人
  - 审批模式
    - 依次审批
    按顺序逐个审批，第1人通过后才轮到第2人，适合需要层级把关的场景。
    - 会签审批
    所有审批人同时收到审批请求，全部通过后文件才会发布，适合需要多方确认的场景。
- 成员管理
  - 成员列表  --  表格  --  姓名 + 权限 + 来源标签 + 操作
  - 来源标签  --  badge  --  「组负责人」=不可删除不可改权限；「飞书同步」=可删除可改权限
  - 权限下拉  --  select  --  管理员 / 可编辑 / 上传下载（组负责人锁定为管理员）
  - 添加成员  --  飞书成员选择器 + 权限下拉 + 添加按钮  --  默认权限 ：上传下载
  - 移除成员  --  文字按钮
- 基本设置
  - 组名称  -- 输入框  --  可编辑，保存时需校验唯一性
  - 归属层级  -- 只读  -- 仅展示，不可编辑
  - 描述  -- 多行输入  -- 可编辑0-200字符
  - 创建时间  -- 只读  -- YYYY-MM-DD
  - 删除组，组下有文件时不可删除，且点击需要有弹窗强提醒进行二次确认
- 特殊规则说明
  - 系统管理员（=总经理）= 公司层负责人，自动继承到公司层下所有子孙组（管理员级，不可手动删除/降权）

**按部门**

（1）定义与定位
按部门分类用于各部门管理内部文档，如部门规章制度、会议纪要、学习分享等；
部门是从飞书组织架构单向同步的管理单元；
部门下支持无限层创建组；
DocFlow 侧不可手动增删部门；

（2）可见范围
- 本部门人员可见。"可见"不等于"有权限"，操作需组内成员权限。
- 部门组也可向其他部门人员开放权限（不限于本部门成员）。
- 部门下的组及组内文档：仅该组的成员（管理员/可编辑/上传下载）可见
- 部门管理员：可以创建组，且只能看到自己创建的子组
- 部门负责人：可以创建组吗，且自动继承到所有管理员创建的子组（管理员级），因此可见所有子组

（3）创建与管理权限
- 在部门下创建组     --     部门负责人、部门管理员
- 管理部门              --     部门负责人
- 创建子组              --     部门负责人（作为继承的管理员）、部门管理员（仅自己是负责人的组下）、组负责人（本组下）
- 重命名                 --     部门负责人（作为继承的管理员）、组负责人（本组）
- 成员管理             --     部门负责人（作为继承的管理员）、组负责人（本组）
- 组设置                 --     部门负责人（作为继承的管理员）、组负责人（本组）
- 删除部门             --     部门负责人

（4）负责人确定方式

系统自动创建部门，关联飞书部门负责人为部门管理员。一个部门可有多个管理员。
- 部门负责人 A：由飞书同步（飞书部门主管字段）；飞书无主管字段时由系统管理员指派
- 部门管理员 B/C：由部门负责人 A 在「部门管理」-「管理员」页面中指定
- B 创建子组 T → B 自动成为 T 的组负责人
- T 的成员表自动继承部门负责人 A（权限=管理员，不可手动删除/降权）
- T 下的子子组 T1（B 建）→ 继承 A + T 的负责人 B

（5）部门管理

触发方式：部门节点 ··· 菜单 -> "管理部门"；或选中部门 → 右侧页面标题后「部门管理」按钮
- 基本信息
  - 部门名称，输入框，不可编辑
  - 描述，多行输入框，可编辑
- 管理员
  - 管理员列表，每行：头像 + 姓名 + 邮箱 + 添加时间 + 角色 badge（部门负责人/管理员）+ 移除按钮
  - 角色 badge，标签，部门负责人 = 红色 badge；管理员 = 主色 badge
  - 添加管理员，操作按钮，打开飞书成员选择器，选人后 toast「已指派 {姓名} 为部门管理员」
  - 移除，操作按钮，confirm「确定移除管理员 {姓名}？」； 约束：若该管理员仍是某子组的组负责人，弹窗「{姓名} 当前是 {组名} 的组负责人，请先移交组负责人后再变更权限」阻塞移除
  - 管理员和负责人可以创建部门下的组，一个部门可以有多个管理员。
- 下属组
  - 创建组，操作按钮，打开创建组弹窗
  - 下属组列表，列：组名称（可点击跳转）/ 负责人 / 文件数 / 成员数 / 子组数 / 更新时间 / 操作（查看按钮）

（6）组设置

触发方式：部门下各组节点 ··· 菜单 -> "管理组"。
结构与公司层的组设置完全一致（审批流配置 / 成员管理 / 基本设置三个 TAB，同时有弹窗入口和页面内 TAB 入口），以下仅列出差异点：
成员管理差异
- 继承成员标签，部门负责人 A 自动出现在列表中，标签「部门负责人（不可移除）」红色 badge
- 继承成员权限，权限=管理员，不可手动删除/降权，权限下拉和移除按钮均 disabled
- 继承范围，部门负责人 A 继承到所有子孙组；T 下子子组 T1 继承 A + T 的负责人 B
（7）特殊规则说明
- 飞书同步方向：部门结构/人员/负责人从飞书单向同步到 DocFlow，DocFlow 不可反向修改
- 飞书新部门同步：自动创建部门节点；有"主管"字段同步为部门负责人，否则空置由系统管理员指派
- 飞书人员移除（离职）：自动清空该人在所有组的成员身份；若是组负责人则自动转给部门负责人 A；触发 离职文档交接流程
- 部门负责人 A 换人：所有子孙组中继承的 A 身份做替换（旧 A 自动移除，新 A 自动加入）
- 飞书同步员工账号策略：全部建 DocFlow 账号（含未加入任何组的人）
- 组负责人离职未主动交接：组负责人自动变更为部门负责人 A

**按产品线**

（1）定义与定位

按产品线分类用于各业务线管理项目文档。产品线下的组称为"项目组"，是最小的成员管理单位。当前业务产品线包括：全球代理IP、zhizhuip、adsl cat、动态资源项目、内部工具。
产品线是 DocFlow 自建的业务分类维度，用于按业务线/项目组织文档。产品线下辖项目组（= 组，仅名称不同），项目组支持无限层级嵌套。

（2）可见范围
- 产品线下的项目组及组内文档：仅该组的成员可见
- 产品线管理员：只能看到自己创建的子组（与部门管理员对称）
- 产品线负责人：自动继承到所有子孙项目组（管理员级），因此可见所有子组

（3）创建与管理权限

创建产品线     --     系统管理员（系统管理员拥有最高权限，下面就不反复提及了）
创建产品线下一级项目组     --     产品线负责人、管理员
创建子组        --     产品线负责人、管理员（仅自己是负责人的组下）、组负责人（本组下）
重命名             --     产品线负责人、组负责人（本组）
成员管理         --     产品线负责人、组负责人（本组）
组设置             --     产品线负责人、组负责人（本组）
交接负责人     --     产品线负责人、组负责人（本组）
删除项目组    --     产品线负责人
删除产品线     --    产品线负责人

> 产品线管理员与部门管理员对称——不进子组成员表，只能管理自己创建的组（通过成为组负责人间接获得权限）

（4）负责人确定方式
- 产品线负责人：由系统管理员在「创建产品线」弹窗中指定
- 产品线管理员：由产品线负责人在「产品线管理」页面中指定
- 子组负责人确定方式与部门侧对称：管理员创建子组 → 自动成为该组负责人 → 产品线负责人自动继承到所有子孙组（管理员级）

（5）创建产品线

入口： 左侧树 hover「按产品线」节点 → + 创建产品线
- 产品线名称，必填，需校验唯一性，1-50字符
- 描述，非必填，0-200字符，placeholder「简单描述本产品线...」
- 添加成员，可选添加初始成员，飞书成员选择器
- 默认成员权限：管理员、可编辑、上传下载（默认选中）
- 确认创建，按钮，点击校验通过后创建产品线，toast「产品线已创建」，刷新左侧树
- 取消，按钮，点击关闭弹窗

（6）产品线管理

触发方式：产品线节点 ··· 菜单 -> "产品线管理"。
- TAB1-基本信息
  - 产品线名称（可编辑，保存时需校验唯一性）
  - 描述（可编辑）
  - 保存，按钮，保存成功 toast「已保存」；取消返回文件列表
  - 删除产品线，按钮，产品线下有组时按钮 disabled + 提示「该产品线下有 N 个组，删除前需先迁移或删除所有组」+ 红色文字「请先迁移下属组」；无组时 confirm 确认后删除
- TAB2-管理员
  - 管理员列表，每行：头像 + 姓名 + 邮箱 + 添加时间 + 角色 badge + 移除按钮
  - 角色 badge，标签，产品线负责人 = 红色 badge；管理员 = 主色 badge
  - 添加管理员，操作按钮，打开飞书成员选择器，选人后 toast「已指派 {姓名} 为部门管理员」
  - 移除，操作按钮，confirm「确定移除管理员 {姓名}？」； 约束：若该管理员仍是某子组的组负责人，弹窗「{姓名} 当前是 {组名} 的组负责人，请先移交组负责人后再变更权限」阻塞移除
  - 管理员和负责人可以创建部门下的组，一个产品线下可以有多个管理员。
- TAB3-下级项目组
  - 创建组，按钮，点击打开创建组弹窗
  - 项目组列表，字段：项目组名称、描述、负责人、文件数、成员数、子组数、更新时间，操作（管理/删除）
  - 管理按钮，点击展示项目组管理弹窗页

（7）组设置

触发方式：产品线下各项目组节点 ··· 菜单 -> "管理组"。
结构与公司层/部门的组设置完全一致（审批流配置 / 成员管理 / 基本设置三个 TAB），差异点：
成员管理差异：
- 继承成员标签：产品线负责人自动出现在列表中，标签「产品线负责人（不可移除）」红色 badge
- 继承成员权限：权限=管理员，不可手动删除/降权，权限下拉和移除按钮均 disabled

（7）特殊规则说明
- 产品线为自建，不从飞书同步（区别于部门）。创建/删除/改名均在 DocFlow 内完成
- 产品线负责人换人：所有子孙项目组中继承的身份做替换（旧负责人自动移除，新负责人自动加入）
- 产品线管理员与部门管理员完全对称：不进子组成员表、只能看到自己建的子组、可创建子组并自动成为该组负责人
- 项目组 = 组，仅名称不同

#### **6.3.5在线编辑器**

**（1）功能概述**
基于 Markdown 的所见即所得在线编辑器，飞书风格 UI（顶栏 + 正文 + 右侧批注 + 左侧目录），支持新建/编辑草稿/编辑已发布副本三种模式，全局走「编辑副本」机制，不直接修改已发布版本。
**（2）三种入口模式**

| **入口模式** | **触发位置** | **行为** |
|---|---|---|
| 新建文档 | 个人中心「+ 新建文档」按钮 | 在个人中心创建一份草稿，文件名「未命名文档」可改 |

1. 审核中文档不可编辑，须先撤回

#### **6.3.7多人协作编辑**
**（1）功能概述**
DocFlow 编辑器支持多人实时协同编辑同一份文档，基于 OT/CRDT 算法保证最终一致性，无需手动解决冲突。一文档一房间（Edit Room），房间内成员共享正文、评论、批注、历史快照。
**（2）协同房间**

| **元素** | **逻辑描述** |
|---|---|
| **房间定义** | - 一文档一房间，同一文档（无论是个人草稿还是编辑副本）只对应一个房间 - 房间寿命，文档存在期间持续存在，文档发布/删除时销毁 - 共享内容，正文 DOM / 行内批注 / 历史快照 / Presence 信息 |
| **加入房间的方式** | - 编辑组内已发布文档（编辑副本），组内拥有可编辑权限的成员 - 在个人中心打开自己的草稿/编辑副本，文档创建者 - 通过链接分享获得可编辑权限，任意组织内部用户（即使不是组成员） - 飞书邀请协作，创建者在飞书中邀请的协作者 |

**（3）实时同步规则**

| **内容** | **同步频率** | **说明** |
|---|---|---|
| 正文文字增删 | 实时（< 200ms 延迟） | 字符级别的 OT/CRDT 同步 |
| 段落格式 | 实时 | 标题/列表/引用等结构变更 |
| 批注新增/回复/解决 | 实时 | 走批注同步通道，独立于正文 |
| 光标位置 | 100ms 节流 | 性能考虑，不实时 |
| Presence 上下线 | 5s 心跳 | 心跳超时即标记离线 |

**（4）协同邀请（分享）**

| **元素** | **逻辑描述** |
|---|---|
| **入口** | 「分享」按钮 → 选择「可编辑」权限 → 复制链接发给协作者 |
| **草稿发布后协作者权限** | - **组内成员** 自动获得「可编辑」权限 - **组外人员** 个人中心保留对该文档的编辑权限，但不在共享文档中可见 |

#### **6.3.8链接分享**

任何用户对自己有访问权限的文档，都可生成分享链接并指定权限（可阅读 / 可编辑），链接仅限组织内部飞书 SSO 用户使用（能登本系统）。

#### 6.3.9选字批注

**（1）功能概述**
用户可在 Markdown 文档中选中任意文字添加批注，批注高亮显示在正文中，右侧面板展示批注列表，支持回复、标记解决。批注绑定具体版本，新版本发布后旧版本批注自动冻结。
**（2）适用范围**

| **入口** | **文档类型** |
|---|---|
| 编辑器（renderEditor） | 草稿 / 编辑副本 |
| 文件详情页（file-detail） | 已发布文档 |
| 全屏文件查看器 | 已发布文档 |
| 审批中心文件预览 | 审批中文档 |

**（3）添加批注**

| **元素** | **逻辑描述** |
|---|---|
| **Step 1：选中文字** | - 触发，在正文区域用鼠标选中一段文字 - 长度限制，1 < 字符数 < 500，超过 500 字不弹浮层 - 适用区域，仅在正文容器内生效 |
| **Step 2：选择浮层** | 选中后弹出微浮层 - 浮层位置，在选区上方 45px，自动避开屏幕边缘 - 按钮，「添加批注」（单一按钮） - 自动隐藏，10 秒未操作自动消失 - 关闭方式，点击其他地方 |
| **Step 3：批注输入面板** | 点击「添加批注」后弹出输入面板 - 引用文字，显示选中的文字（超过 100 字截断为「...」） - 批注内容，多行文本框，必填 - 1-1000 字符，去除首尾空格 - autofocus，placeholder「输入批注内容...」 - 取消，关闭面板，不保存 - 提交批注，保存批注、关闭面板，正文中对应文字加上高亮，右侧批注面板自动展开并定位到新批注，toast「批注已添加」 |
| **异常处理** | - 批注内容为空，toast「请输入批注内容」并保留焦点 - 网络异常，toast「批注保存失败，请重试」并保留输入面板 |
| **批注高亮（正文中）** | - 样式，选中文字背景为浅黄色，下方加 2px 黄色实线 - 点击高亮，右侧批注面板自动展开，并滚动到对应批注卡片 - 多个批注重叠，同一段文字有多条批注时，以最早的批注为主样式，hover 显示数字角标 |

**（4）右侧批注面板（侧边栏）**

| **元素** | **逻辑描述** |
|---|---|
| **入口** | - 编辑器，可通过右上角 × 收起 - 文件详情页，默认收起，点击「批注」按钮或文字高亮展开 - 审批中心，默认展开（审批人进入时批注面板自动展开） |
| **面板布局** | - 标题栏，「批注」+ 数量 + 右侧 × 关闭按钮 - 筛选 TAB，「全部 (N)」「未解决 (N)」「已解决 (N)」三按钮切换 - 批注卡片列表，按时间倒序排列 |
| **批注卡片字段** | - 作者头像，颜色由作者 ID 派生或同步飞书头像 - 作者姓名 - 时间，HH:MM / 「昨天 HH:MM」 / `YYYY-MM-DD` - 引用文字，灰色斜体引号包裹「"...选中文字..."」 - 批注内容，文本 - 回复区，已有回复时显示，每条回复含小头像 + 姓名 + 内容 + 时间 - 操作按钮，回复/标记解决/删除 - 回复/删除可参考飞书批注 - @某人 自动消息通知 |

**（5）批注生命周期规则**
1. **完整规则表**

| **场景** | **批注行为** |
|---|---|
| 草稿编辑中 | 协作者可添加 / 回复 / 标记解决批注，实时同步 |
| 草稿 → 提交审批 | 批注随文档进入审批，审批人可见 |
| 审批过程中 | 审批人可新增批注（作为修改意见） |
| 审批通过 → 发布 | 草稿期间和审批期间的批注成为该版本的正式批注 |
| 审批驳回 | 审批人的批注保留在草稿上，供编辑者参考 |
| 已发布文档 | 组成员可添加 / 回复 / 标记解决批注 |
| 创建编辑副本 | 不继承上一版本批注，从零开始 |
| 创建编辑副本时 | 弹窗提示「当前版本有 N 条未解决批注」（仅提示不阻断） |
| 新版本发布后 | 旧版本批注冻结（不可新增 / 回复 / 标记解决 / 删除） |
| 查看历史版本 | 批注面板顶部显示「⚠ 该版本批注已冻结」标识，仅只读 |
| 冻结不可逆 | 即使新版本被删除 / 回滚，旧版本批注仍冻结 |

2. **冻结状态视觉区分**

| **元素** | **已冻结样式** |
|---|---|
| 整个批注面板 | 顶部黄色横幅「⚠ 该版本批注已冻结，不可编辑」 |
| 批注卡片 | 整体灰度 60% |
| 操作按钮 | 全部隐藏（无回复 / 解决 / 删除） |
| 高亮 | 正文高亮保留但 hover 提示「（已冻结）」 |

3. **编辑副本创建时的未解决批注提示**
参考编辑副本确认弹窗，在弹窗中追加一行：
`⚠ 当前版本有 {N} 条未解决批注（不会带入编辑副本）`
仅提示，不阻断创建。
#### 6.3.10归属人转移
**（1）功能概述**
文档归属人因调岗等原因不再需要管理该文档时，可主动将归属权转给他人。新归属人需「同意认领」后才正式生效，转移后已发布的不需要重新审批。
**（2）入口**
- 个人中心 → 我创建的 → 操作栏 ··· 更多 → 「转移归属人」
- 编辑器顶栏 ··· → 「转移归属人」
- 文件详情页（个人中心入口）→ 操作栏 ··· → 「转移归属人」
**（3）转移说明**

| **元素** | **逻辑描述** |
|---|---|
| **弹窗字段** | - 文档，只读文本 - 新归属人，飞书选人器（单选） - 审批中心，默认展开（审批人进入时批注面板自动展开），点击输入框打开「飞书成员选择器」 - 选择范围，全员 - 选择数量，单选 - 排除项，不能选择当前归属人自己 - 取消，关闭弹窗 - 发送转移请求，创建一条 pending 状态的转移请求记录 → 通过飞书机器人向新归属人推送卡片消息 → toast「已发送转移请求至「{姓名}」，等待对方确认」→ 关闭弹窗 |
| **飞书卡片消息** | ```[卡片标题]：归属人转移请求 {发起人姓名} 请求将文档《{文档名}》的归属人转移给您 文档所在组：{组名} / {部门 or 产品线} 当前版本：v{X.Y} [同意] [拒绝] ``` |

**（4）接收方处理**

| **元素** | **逻辑描述** |
|---|---|
| **同意** | - 后端更新文档归属人为新接收人，状态变为 accepted - 消息通知双方 - 文档在原归属人的「我创建的」TAB 中消失，进入新归属人的「我创建的」TAB - 原归属人若仍属于该组的成员，保留相应权限；若不在组内，则失去访问 - 卡片消息更新为「✓ 已同意，归属权已转移 |
| **拒绝** | - 更新请求状态为 rejected - 通知发起方「{接收人} 已拒绝转移请求」 - 归属权不变，发起方需重新发起或选择其他人 - 卡片消息更新为「✗ 已拒绝」 |
| **长时间未响应** | - 阈值，3 天未点击同意/拒绝 - 系统自动关闭请求（status = expired），通知发起方「转移请求已过期，请重新发起」 |

**（5）待响应阶段的文档可用性**

| **操作** | **是否可用** |
|---|---|
| 文档查看 | ✅ |
| 文档编辑（创建副本） | ✅ |
| 提交审批 | ✅ |
| 删除文档 | ❌ 暂不允许（避免转移目标失效） |
| 再次发起转移 | ❌ 不允许 |

<quote-container>
业务约束：待响应期间不锁定文档，正常使用，但不允许影响归属权的危险操作。
</quote-container>

**（6）不做清单**
- 不做多选转移（一次只能转一个文档给一个人）
- 不做转移后重新审批
- 不做转移过程中的文档锁定
- 不做转移撤销

| 部门管理员 | 由 A 指定 | 部门负责人(A)指派 | 修改部门信息、创建一级子组（自动成为组负责人）、只能看到自己创建的子组 |

### **6.10 文档引用**
#### **6.10.1功能概述**
解决同一文档需要在多个组中可见但只能单组归属的问题。通过「引用」机制，组管理员可将其他组的已发布文档映射到本组文件列表中，引用文档为只读映射，源文档更新后目标组自动同步最新版本。
**核心概念：** 引用 = 指针，不是副本。文档始终只有一个归属组（源组），引用组看到的始终是源组的最新已发布版本。
**功能路径：**侧边栏 → 共享文档 → 选择任意组 → 操作栏「添加引用」按钮
#### **6.10.2文档列表说明**

| **区域/元素** | **类型** | **逻辑描述** |
|---|---|---|
| 文件名列 | 文字+图标 | 显示源文件名 + 🔗引用图标（SVG 线性链接图标，主色调），hover 引用图标显示气泡提示「引用自：{源组名}」 |
| 行背景 | 样式 | 引用文档行使用浅蓝背景（#fafcff），与普通文档行视觉区分 |
| 版本列 | 文字 | 实时显示源文档的当前版本号（非引用时的快照版本） |
| 状态列 | 标签 | 始终显示「已发布」（只有已发布文档才能被引用） |
| 上传者列 | 文字 | 显示源文档的上传者 |
| 更新时间列 | 文字 | 显示源文档的最后更新时间 |
| 操作列 | 按钮 | 见下方功能详述 |
| 收藏列 | 按钮 | 支持收藏，行为与普通文档一致 |
| 复选框 | 禁用 | 引用文档的复选框为 disabled 状态（不参与批量操作） |
| 操作列按钮 |  | **详情** - 可见角色：所有组成员 - 点击行为：跳转到引用文档详情页 **“...”更多** - 可见角色：组管理员 - 点击行为：展开下拉菜单，仅含「取消引用」（红色文字） --- **与普通文档操作栏对比** - 普通文档：编辑、下载、置顶/取消置顶、文档级权限、跨组移动、移除 - 引用文档：下载、取消引用 |

#### **6.10.3添加引用弹窗说明**

**（1）入口**
- 组详情页操作栏 - 「🔗 添加引用」按钮
- 可见角色：组管理员

| **字段** | **类型** | **说明** |
|---|---|---|
| 组筛选 | 下拉选择器 | 默认"所有可见组"，选项为当前管理员有权限查看的所有组（排除当前组本身），显示格式「组名（N个文件）」 |
| 文件名搜索 | 输入框 | placeholder="搜索文件名..."，输入后实时筛选文档列表 |
| 已选计数 | 文字 | "已选择 N 个文档"，实时更新 |
| 文档列表 | 可选列表 | 可滚动；展示管理员可见的所有已发布文档 **文档列表项说明** - 选中指示器，复选框 - 文件名 - 元信息，{组名} · {版本} · {上传者} · {更新日期} - 已引用标签，如该文档已被当前组引用，显示橙色标签「已引用」，该项不可点击选择 |

| 选择的文档已被引用 | 不可选（UI 层面已禁止，标「已引用」） |
| 源文档在弹窗打开期间被移除 | 确认时后端校验，失败文档跳过，成功的正常引用，提示「N 个文档引用成功，M 个文档已失效」 |

#### **6.10.4取消引用**
**入口 ：** 文件列表 → 引用文档行 → 更多菜单「···」→「取消引用」（红色文字）

| **元素** | **说明** |
|---|---|
| 提示文案 | "确定要取消对以下文档的引用吗？" |
| 文档信息卡片 | 灰色背景区域，显示文件名和来源（"引用自：{源组名}"） |
| 补充说明 | "取消引用后，该文档将从本组文件列表中移除。不会影响源文档和其他组的引用。" |

#### **6.10.5引用文档详情页**
**(1) 入口**
文件列表 → 引用文档行 → 点击「详情」按钮
**(2) 页面说明**
引用文档详情页复用现有文件详情页（§6.3.4）的整体布局，但进行以下差异化处理：

| **区域/元素** | **与普通文档的差异** |
|---|---|
| 顶栏标题 | 文件名后显示 🔗 链接 SVG 图标 |
| 只读提示条 | 顶栏下方新增蓝色提示条：「🔗 这是一个引用文档（只读），源文档归属于 **{源组名}**」 |
| 操作按钮 | 仅显示「下载」按钮，不显示编辑/分享/更多菜单 |
| 返回按钮 | 返回引用所在的目标组文件列表（非源组） |
| 文档预览 | 与普通文档一致，展示源文档的最新已发布版本正文 |
| 批注/评论 | 展示并可参与源文档的批注和评论 |
| 版本记录 | 展示源文档的版本历史 |

**(3) 文档级权限保留规则**

| **场景** | **行为** |
|---|---|
| 用户通过分享获得了源文档的**可编辑**权限 | 详情页按可编辑权限展示（显示编辑按钮等），不受引用组只读限制 |
| 用户通过协作邀请加入源文档 | 同上，文档级权限优先于引用组默认权限 |
| 用户无任何文档级权限 | 按引用组默认权限展示：只读+下载 |

**优先级：** 文档级权限（分享/协作）> 引用组默认权限（只读）> 无权限
#### **6.10.6引用自动失效**
**（1）触发条件**

| **触发场景** | **说明** |
|---|---|
| 源文档被管理员从源组「移除」 | 文档退回归属人个人中心，不再处于已发布状态 |
| 源文档被归属人在个人中心「删除」 | 文档进入回收站 |

<quote-container>
边界澄清：编辑副本/审批中不影响引用，因为已发布版本仍然存在。只有文档从组中彻底移除（退回个人中心或删除）时引用才断裂。
</quote-container>

**(2) 失效行为**
1. 系统检测到源文档不再处于已发布状态
1. 自动删除所有关联的引用关系
1. 引用文档从所有目标组的文件列表中消失
1. 触发飞书通知（见 §6.10.7）
#### **6.10.7引用失效飞书通知（M25 新增）**

| **属性** | **说明** |
|---|---|
| 编号 | M25 |
| 消息类型 | 引用失效通知 |
| 触发场景 | 源文档被移除/删除，导致引用自动断裂 |
| 接收人 | 每个目标组的管理员（每个引用组的管理员各收一条） |
| 卡片按钮 | - |
| 消息模版 | 文档《{文件名}》已从组《{源组名}》移除，您的组《{目标组名}》中的引用已自动取消 |

**分类归属：** 归入「文档生命周期类」（与 M8/M9 并列）
#### **6.10.8 引用文档置顶/收藏**
行为规则与普通文档完全一致（参见 §6.3.3 文件列表中的置顶/收藏说明）

- 收藏：所有成员可收藏引用文档，收藏后在个人中心「个人收藏」TAB 中展示

#### **6.10.10 业务规则**

| **规则** | **说明** |
|---|---|
| 引用范围 | 管理员可引用自己有权限查看的所有已发布文档（不限同部门/同产品线） |
| 版本同步 | 引用 = 指针，始终实时展示源文档的最新已发布版本，不是引用时的快照 |
| 归属不变 | 引用不改变文档归属关系，文档始终只有一个归属组 |
| 导航不变 | 引用不影响导航可见性——B 组成员不会因引用而看到 A 组导航节点 |
| 不计入统计 | 引用文档不计入目标组的文档数量统计（但在翻页器中标注「含 N 条引用」） |
| 不参与批量 | 引用文档的复选框为禁用状态，不参与批量跨组移动/批量删除 |
| 引用去重 | 同一文档只能被同一组引用一次，重复引用时 UI 层面标「已引用」不可选 |

#### **6.10.11 不做清单**

| **不做** | **原因** |
|---|---|
| 不做多归属 | 文档始终只有一个归属组，引用只是只读映射 |
| 不做引用组内编辑 | 所有编辑操作回到源组进行，引用组不产生独立的编辑副本 |
| 不做引用组内审批 | 审批流仍在归属组内执行 |
| 不做引用链传递 | B 组引用 A 组文档后，C 组不能从 B 组「引用的引用」 |
| 不做引用权限分级 | 引用组统一只读，不支持在引用时配置权限级别 |
| 不做引用数量限制 | 一个文档可被任意数量的组引用 |
